# 西藏边境线地图动画：项目记忆

本文件是本项目的持久上下文。新会话、上下文压缩或任务交接后，先读本文件，再读与任务相关的专项文档。

## 项目定位

这是一个“冒险旅行交互 Web 视频”Demo：用 MapLibre、卫星影像、GPX 路线、照片和视频讲述 2024 年 12 月的西藏边境旅行。

它不是严格的 GPS 回放工具。目标优先级是：

1. 成片叙事清楚。
2. 镜头稳定、路线可信。
3. 保留真实时间轴。
4. 尽量尊重原始 GPS；原始记录影响观感时允许使用明确的剪辑指令修正。

## 当前入口

- 网页：`地图行驶动画.html`
- 视觉样式：`assets/map-animation.css`
- 全局速度/镜头/底图参数：`assets/map-config.js`
- 播放编排：`assets/map-animation.js`
- 人工剪辑指令：`route_data/route-overrides.json`
- 路线分层说明：`route_data/README.md`、`route_data/data-model.json`
- 导航匹配中间层：`route_data/navigation/matched-route.json`
- 指令 Schema：`route_data/route-overrides.schema.json`
- 生成路线：`route_data/processed/route-data.js`
- 轨迹处理：`scripts/process_route_osm.py`
- 校验：`scripts/validate_route_demo.js`
- A→B→A→B 折返扫描：`scripts/find_route_backtracks.py`

不要手工编辑 `route_data/processed/route-data.js` 或 `.json`；修改 overrides 后用脚本重新生成。

## 路线四层数据架构

### 1. 原始 GPS 层

- 权威源：`2024.01.01-2024.12.31.gpx`，永不修改。
- 规范化只读镜像：`route_data/source/raw-track.json`。
- 每个点带稳定的 `source_id`；`accepted` 只表示是否进入后续处理。

### 2. 人工剪辑层

- `route_data/editorial/exclusions.json`：自动剔除点、人工替换区间及受影响的原始点 ID。
- `route_data/editorial/anchors.json`：手工坐标、必须经过的点、字幕和媒体触发点。
- 权威维护入口仍是 `route_data/route-overrides.json`；上述两个文件由处理脚本生成，不手工编辑。

### 3. 导航道路层

- `route_data/navigation/matched-route.json`：清理原始点并完成 OSM/OSRM 道路匹配后的中间路线。
- `route_data/osm_cache/` 和 `route_data/osrm_drive_cache/`：外部导航几何的本地缓存。
- 该层尚未应用人工 `geometry_edits`，是重新剪辑的稳定基线。

### 4. 动画呈现层

- `route_data/processed/route-data.json`、`.js`：加入人工几何修正、圆角和插值后的最终播放数据。
- `source` 标记点来自 `raw`、`editorial_anchor`、`navigation:osm`、`navigation:osrm` 或 `rendered`。
- 第四层只能由前三层重建，不能作为道路匹配或人工修正的输入。

`route_data/route-overrides.json` 保存所有人为决策：

- `mode_ranges`：步行、慢车、快车。
- `geometry_edits`：删除无意义折返、顺直路线、圆滑转弯。
- `road_corridors`：指定 OSM/OSRM 道路吸附区间。
- `story_events`：统一的新标题/照片/视频事件。
- `camera_directives`：固定区域、柔和跟随、缩放、俯仰和朝向。
- `title_cards`、`supply_events`、`media_events`、`camera_ranges`：旧格式，继续兼容。

人工剪辑意图优先于自动道路匹配。

## 浏览器运行层

浏览器模块通过 `window.RouteDemo` 协作，无需打包工具：

- `assets/modules/route-model.js`：距离、时间、速度、转弯角、GPS 最近点。
- `assets/modules/directive-engine.js`：把时间或坐标编译成路线触发点。
- `assets/modules/camera-policy.js`：固定区域、柔和跟随、俯仰、bearing、look_at。
- `assets/modules/media-overlay.js`：媒体卡、视频延迟加载、暂停与恢复。
- `assets/modules/coordinate-picker.js`：人工地图选点、GPS 显示和复制。
- `assets/map-animation.js`：MapLibre 图层和播放循环，只做模块编排。

详细接口和配置示例见 `地图动画代码架构.md`。

## 视觉与交互风格

### 地图

- 默认使用卫星影像，因为信息密度高，能看见道路、湖泊、山体和村落。
- “塞尔达/冒险地图底图”方案暂时 hold；不要重新加入低信息量或简单滤镜式冒险底图。
- 冒险感主要来自标题、章节卡、物资卡、字体、装饰线和节奏，不牺牲地图信息量。
- 默认北向，`bearing: 0`，不按照汽车或人物行进方向旋转地图。
- 只有明确的叙事镜头可以使用 `look_at`，例如徒步时面向库拉岗日。

### 镜头

- 快车赶路：全局视角。
- 慢车、步行、城市、景区和盘山路：细节视角。
- 镜头稳定优先于主体居中；车和人物可以在画面中移动。
- 盘山路、湖边等区间允许固定区域镜头，禁止频繁放大缩小或追车抖动。
- 柔和跟随时镜头移动要慢于主体，使用较低 `follow_factor`。
- 快慢模式切换必须渐变，不能突然跳 zoom；切入细节时同步降低播放速度。
- 城市步行可以放得更大，但人物播放速度必须更慢。
- 盘山路播放速度要慢，转弯越急越慢。

### 路线

- 路线必须尽量落在真实道路上；车辆掉进湖、穿山或长距离直线时应做 OSM/OSRM 匹配或人工修正。
- GPS 折返、路口徘徊、原路上下且无叙事意义的支线应删除。
- 删除支线时保留起止真实时间，只替换中间几何；不要压缩时间轴。
- 转弯使用多个缓和点，避免尖角。修改后检查最大转角和反向步数。
- 轨迹可以比真实记录直接，但不要为了直而穿越建筑、湖泊或明显不可通行地形。

### 人物与车辆

- 步行图标使用 emoji 风格，不使用写实大图标。
- 三人排成两排：前排黑衣男，后排红衣情侣；整体保持小尺寸。
- 快车、慢车、步行在 HUD 中不显示模式文字，显示实际 `xx km/h`。

### 标题与媒体卡

- 地点字幕和媒体卡采用克制的冒险风：深青黑半透明底、金色细线、浅青点缀、宋体标题、切角轮廓。
- 不照搬游戏 UI，不堆砌符文；卫星地图仍是主体。
- `chapter`：关键地点，暂停路线，居中显示。
- `memory`：普通记忆，右下角轻量显示，不暂停路线。
- 视频默认只显示 poster；点击后才加载 H.264/AAC MP4。
- 播放视频时暂停行驶，展开为电影屏幕大小但不强制全屏；关闭后恢复。
- 右下角地图 attribution/i 图标保持隐藏。
- 左下角控制面板保持紧凑，不恢复大面积图例。
- 进度条占满控制面板整行，使用 100000 级精度，便于定位具体时间段。
- “📍 选点”用于让用户直接点击地图确认剪辑坐标；点击后显示并复制 `[经度, 纬度]`，不要再仅凭截图猜位置。

## 常用工作流

只修改标题、媒体或镜头事件：

```bash
python3 scripts/process_route_osm.py --events-only
```

只修改人工轨迹几何：

```bash
python3 scripts/process_route_osm.py --geometry-edits-only
```

需要重新做道路匹配时：

```bash
venv-mv/bin/python scripts/process_route_osm.py --no-driving-match
```

每次修改后：

```bash
node scripts/validate_route_demo.js
```

高风险镜头、媒体暂停、路线跳转需要再用真实 Chrome 回归，不能只做语法检查。

## 修改纪律

- 用户说“看一下/诊断”时先解释原因，不自动扩大修改范围。
- 用户指定一个时间段时，只修改该区间，并检查前后连接。
- 路线和事件配置应数据驱动；不要把某个地点的经纬度或时间硬编码回播放循环。
- 镜头策略写入 `CameraPolicy` 和 `camera_directives`，不要在 `setDist` 继续堆特殊地点判断。
- 新标题/媒体优先使用 `story_events`；旧事件保持兼容，不必一次性迁移。
- 保留用户已有素材和未关联修改，不做破坏性清理。
- 当前目录可能不是 Git 仓库；不要假设可以用 git 回滚。
