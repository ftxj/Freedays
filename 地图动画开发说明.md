# 地图动画开发说明

跨会话或上下文压缩后，先阅读根目录的 [AGENTS.md](AGENTS.md)，其中记录当前设计原则、基线和不可回退的决定。

## 文件职责

- `地图行驶动画.html`：页面结构和资源加载顺序，不放业务逻辑或轨迹数据。
- `assets/map-animation.css`：HUD、控制面板、地点字幕、物资/动态记忆卡和视频窗口样式。
- `assets/map-config.js`：播放速度、镜头缩放、过渡时长和底图风格。
- `assets/map-animation.js`：地图图层、镜头、播放状态和交互事件。
- `assets/modules/route-model.js`：路线距离、时间、速度和转弯计算。
- `assets/modules/directive-engine.js`：把 GPS/时间指令编译为路线触发点。
- `assets/modules/camera-policy.js`：固定区域、柔和跟随、俯仰和朝向策略。
- `assets/modules/media-overlay.js`：照片/视频弹层与播放暂停管理。
- `assets/modules/coordinate-picker.js`：地图点击选点、GPS 显示和复制。
- `route_data/map-content.js`：人工维护的停靠点、山峰和住宿点。
- `route_data/route-overrides.json`：行驶模式、固定镜头、字幕、物资/媒体事件和轨迹修正。
- `route_data/processed/route-data.js`：由处理脚本生成，不手工编辑。
- `route_data/route-overrides.schema.json`：剪辑指令格式约束。
- `route_data/source/raw-track.json`：原始 GPX 的规范化只读镜像。
- `route_data/editorial/`：剔除点/区间及人工关键点。
- `route_data/navigation/matched-route.json`：OSM/OSRM 匹配后的稳定中间层。
- `route_data/processed/route-data.js`：带来源字段的最终动画插值层。

完整的扩展方式见 [地图动画代码架构.md](地图动画代码架构.md)。

## 常见扩展

### 增加地点字幕、物资或媒体事件

只修改事件时，编辑 `route_data/route-overrides.json` 后复用现有轨迹几何：

```bash
python3 scripts/process_route_osm.py --events-only
```

只有道路或轨迹处理逻辑变化时，才完整重新生成轨迹：

```bash
venv-mv/bin/python scripts/process_route_osm.py --no-driving-match
```

配置新的 `road_corridors` 后，先下载当地 OSM 道路并写入本地缓存：

```bash
venv-mv/bin/python scripts/process_route_osm.py --prepare-road-corridors
```

`force_local: true` 表示该时间走廊优先使用本地 OSM 道路图；`max_snap_m`、`max_route_m` 和 `max_route_ratio` 共同阻止错误吸附和异常绕路。人工 `geometry_edits` 最后应用，优先级高于道路匹配。

跨越多个稀疏 GPS 锚点的复杂道路重建使用 `geometry_edits[].path_id` 引用 `route_data/editorial/manual-paths.json`。路径由 `scripts/build_editorial_road_paths.py` 生成，普通路线构建不访问网络。

只下载或更新指定的人工道路路径：

```bash
python3 scripts/build_editorial_road_paths.py --ids d26_road_1120_1300_review
```

`route_connections` 用于住宿点到次日出发点这类“需要画线，但不应参与车辆播放和时间插值”的跨日接驳。网页以较淡的虚线绘制，真实时间和住宿状态不受影响。

只增加人工轨迹顺直/去折返修正时，不重新匹配道路：

```bash
python3 scripts/process_route_osm.py --geometry-edits-only
```

该命令从 `navigation/matched-route.json` 重建最终层，多次执行不会累积修正。不要把 `processed` 中的插值点回灌到原始或导航层。

`media_events` 的 `priority` 分为两级：

- `chapter`：暂停行驶，在画面上方显示重要章节卡。
- `memory`：不暂停行驶，在右下角显示轻量动态记忆卡。

视频使用 H.264/AAC MP4。页面只预载封面，用户点击后才加载视频文件。

一个地点有多段视频时使用 `clips`，卡片会显示 `01 / 04` 和左右切换按钮：

```json
{
  "id": "yamdrok-winter-memory",
  "lonlat": [90.858712, 28.875549],
  "title": "冬日羊卓雍措",
  "priority": "chapter",
  "pause": true,
  "clips": [
    {
      "video": "video/web/pumoyongtso-aerial.mp4",
      "poster": "video/web/pumoyongtso-aerial.jpg",
      "title": "冰湖航拍",
      "label": "播放羊卓雍措航拍"
    }
  ]
}
```

选中的原始视频由 `scripts/build_map_video_previews.sh` 生成网页预览版，脚本不会修改原片。媒体卡可在不加载地图的情况下单独预览：

```text
http://127.0.0.1:8765/tools/media-card-preview.html?media=yamdrok-winter-memory
```

### 增加地图风格

1. 在 `assets/map-config.js` 的 `mapStyles` 中增加配置。
2. 在 `地图行驶动画.html` 的底图下拉框增加同名 `option`。

### 调整速度或镜头

只修改 `assets/map-config.js` 中的 `playback` 或 `camera`，不要在播放主逻辑中散落常量。

## 修改后校验

```bash
node scripts/validate_route_demo.js
```

该脚本会检查脚本语法、资源加载、轨迹点数量、地图内容、底图配置以及物资/媒体文件。
